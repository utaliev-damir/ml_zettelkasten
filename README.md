# Структура знаний

Это рабочий репозиторий для подготовки к ШАД и изучения Machine Learning. Здесь собраны 9 типов сущностей, каждая лежит в своей папке и имеет свой шаблон в `_templates/`.

Репозиторий представляет собой **хранилище Obsidian (vault)**. Чтобы работать с ним как задумано — со связями между заметками, графом и шаблонами — открывать его нужно именно в Obsidian, а не как набор отдельных `.md`-файлов.

## Как начать работать с хранилищем

1. Склонировать репозиторий на свой компьютер (`git clone`).
2. Открыть [Obsidian](https://obsidian.md/) → `Open folder as vault` → выбрать папку этого репозитория.
3. Включить нужные community-плагины (см. раздел «Плагины»).
4. В настройках указать папку шаблонов — `_templates/` (см. раздел «Шаблоны»).


## Типы заметок

|Тип|Папка|Что это|
|---|---|---|
|Определение|`Определения/`|Математическое определение объекта|
|Свойство|`Свойства/`|Математическое свойство|
|Теорема|`Теоремы/`|Теорема с доказательством|
|Метод|`Методы/`|Подход к решению класса задач|
|Алгоритм|`Алгоритмы/`|Конкретный алгоритм с псевдокодом и сложностью|
|Сигнал|`Сигналы/`|Признак условия, указывающий на метод или тип задач|
|Структура данных|`Структуры и типы данных/`|Тип данных и операции над ним|
|Теория алгоритмов|`Теория алгоритмов/`|Теоретические понятия (асимптотика, сложность)|
|Задача|`Задачи/`|Решённая задача с разбором|

## Поле «формат» у задачи

|Значение|Когда использовать|
|---|---|
|`доказательство`|Доказать, проверить, обосновать|
|`вывод`|Получить формулу или оценку — например, вывод производной|
|`численная`|Посчитать конкретное значение|
|`код`|Реализовать алгоритм, написать функцию|

## Служебные папки

Помимо папок с типами заметок, в хранилище есть служебные:

|Папка|Назначение|
|---|---|
|`MOC/`|Карты содержания (Maps of Content) — точки входа в крупные темы, откуда расходятся ссылки на заметки|
|`Static/`|Вложения: изображения, схемы и прочие файлы, на которые ссылаются заметки|
|`_templates/`|Шаблоны для каждого типа заметок|

## Шаблоны

Каждому типу заметки соответствует шаблон в папке `_templates/`. Шаблоны подставляются плагином **Templater**. Чтобы они работали, в настройках Templater нужно указать `_templates/` как папку шаблонов:

`Настройки` → `Templater` → поле «Template folder location» → указать `_templates`.

## Плагины

Для корректной работы хранилища нужно установить и включить community-плагины:

- **Templater** — расширенные шаблоны заметок (подстановка шаблонов из `_templates/`).
- **Dataview** — динамические списки и таблицы по заметкам.
- **Metadata Menu** — управление полями метаданных заметок (в том числе поле «формат» у задач).
