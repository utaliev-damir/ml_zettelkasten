Решает класс задач, где нужно разбить большую задачу на более мелкие, решить мелкую задачу. Причем если мелкие задачи повторяются, то мы их не решаем. Пусть нам надо найти некоторое число для ряда входных данных, тогда это будет функция T(n1, n2, nk), и у каждого ряда данных есть подзадачи i1…im, такие что m <=n. Например, если на вход подается массив [1,2,3], то это n, его можно разбить на 1,2,3

Классическим примером задачи для решения ее динамическим программированием это числа Фибоначчи. 
Эту задачу можно решить рекурсивно

### Числа Фибоначчи
1 1 2 3 5 8 13 21
1 2 3 4 5 6  7  8
```python

# на вход дает порядковый номер числа
# на выходе нужно найти число фибоначи
def fib(n):
    if n == 1 or n == 2:
	    return 1
	res = fib(n-1) + fib(n-2)
	return res        
```
В чем проблем такого подхода - мы будет рассчитывать одни и те же значения несколько раз, например для n=5 мы рассчитаем результат для 3 два раза, для 2 три раза. Из за этого кол-во выполняемых операций будет увеличиваться. 

```
5 -> 4 -> 3 - > 2 = 1
			- > 1 = 1	
	   -> 2 = 1
	   
	 3 -> 2 = 1
	   -> 1 = 1	
```

При решении рекурсией мы разбиваем задачи сверху вниз, запоминаем самые нижние результаты и начинаем их возвращать в вышестоящие вызовы.

Динамическое программирование же решает эту проблему - мы разбиваем задачу снизу и двигаемся наверх, сохраняя результаты выполненных операций, чтобы не пришлось их выполнять несколько раз.

Так с задачей Фибоначчи с помощью динамического программирования можно решить ее так:
```python

# на вход дает порядковый номер числа
# на выходе нужно найти число фибоначи
def fib(n):
    if n == 1 or n == 2:
	    return 1
	last = 1
	b_last = 1
    for x in range(3, n+1):
	    last, b_last = last + b_last, last
	return last
		
```
Разница в скорости - разительная.
