Ортогональность - понятие из геометрии, это две перпендикулярные линии или вектора, которые направлены в разные стороны и между ними 90 градусов. 
В проектировании систем это означает что компоненты должны быть независимы, изменения внесенные в один компонент не должны за собою влечь изменения в других компонентах.

В качестве примера ортогональной системы приводится система управления вертолетом - оказание влияния на один из компонентов влечет за собой изменение состояния другого компонента. 

Преимущества ортогональной системы:
- Производительность - за единицу времени можно сделать больше, то есть если нужно изменить один модуль мы потратим время на изменение только этого модуля а не других
- Комбинации независимых компонентов дает возможность создавать новые решения

Когда вношу изменения в функцию или класс нужно спросить себя как много модулей это заденет, ответ один. 

Связано с [[DRY]]

Кодирование
- поддерживать код развязанным - автономность компонентов, внесение изменение не затрагивает другие модули. 
- Не завязываться на глобальных переменных - при переходе на многопоточность потоки будут обращаться к одной переменной, возможно в каком то потоке нужно будет менять значение
- Синглтон - концепция согласно которой экземпляр класса создается в единожды
- Избегать сходных функций - избегать ситуаций когда есть функции у которых к примеру одинаково начало и конец, но отличается центр. Это говорит о том, что происходит дублирование кода, скорее всего нужно выделить отдельную функцию для этого.
