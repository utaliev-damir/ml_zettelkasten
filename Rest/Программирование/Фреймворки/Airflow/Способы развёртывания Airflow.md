Cуществует несколько способов развёртывания Airflow, эти способы определяются тем, где, кем и как будут развернуты [[Компоненты Airflow| компоненты Airflow]].

* Базовый способ - разворачивается на одной машине, одна роль.
	
	*![[Pasted image 20240514200405.png]]
* Распределённый 
	* запуск на нескольких машинах с наличием [[Компоненты Airflow#^c837b0| различных ролей]], которая имеет соответствующий доступ
	* изолированные периметры безопасности для некоторых компонентов
	* Webserver не общается напрямую с [[Компоненты Airflow#^0aa8a2|DAG folder]], а забирает информацию о коде из базы метаданных
	 ![[Pasted image 20240514200456.png]]
* Разделенный
	* запуск на нескольких машинах с наличием [[Компоненты Airflow#^c837b0| различных ролей]], которая имеет соответствующий доступ
	* scheduler отделен от [[Компоненты Airflow#^0aa8a2|DAG folder]], получает данные о ДАГах из базы метаданных
	* за обработку файлов в DAG folder отвечает отдельно выделенный компонент -[[Компоненты Airflow#^eac50a| DAG proccessor]]
