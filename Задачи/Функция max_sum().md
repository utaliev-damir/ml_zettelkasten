## **Условие**  
  
Реализуйте функцию `max_sum()`, которая принимает один аргумент:

- `pairs` – список, элементами которого являются кортежи из двух **положительных** целых чисел $1≤len(pairs)≤10^6$

Функция должна выбирать из каждого кортежа списка `pairs` ровно одно число так, чтобы сумма всех выбранных чисел **не делилась** на `3` без остатка и при этом была максимально возможной. Возвращаемое значение функции – сумма выбранных чисел. Если подходящую сумму получить невозможно, функция должна вернуть значение `-1`.
  
## **Основной метод**  
  
[[ ]]  Перебор по кортежам массива, выбор максимального значения из кортежа. 
  
## **Ключевая идея**  
  
Чтобы сумма была максимальной нужно выбирать наибольшие элементы, но в таком случае мы можем получить сумму, которая будет кратна 3. 

- Как это исправить? В одном из кортежей взять минимальный элемент, а не максимальный. 
- Но как обеспечить при этом максимальность итоговой суммы? Нужно подобрать такой элемент, разница с которым у максимального элемента из того же кортежа будет минимальной. 
- А как обеспечить делимость итоговой сумма на 3? нужно чтобы разница с предыдущего шага была не кратной 3
  
## **План решения**  
  
```python

def max_sum(pairs):
    
    result = 0
    min_diff = float("inf")
    
    for a,b in pairs:
        result += max(a, b)
        diff = abs(a - b)
        if diff%3 != 0 and diff < min_diff:
            min_diff = diff
    
    if result%3 != 0:
        return result
    
    if min_diff == float("inf"):
        return -1
    
    return result - min_diff
	
```
  
## **Ошибки / тупики**  
  
Сначала была гипотеза, что если конечная сумма не должна быть кратна 3, то и промежуточные будут не кратны, но это не так. Также узнал, что число кратно 3, если сумма его цифр кратная трем (из свойств десятеричной системы)
